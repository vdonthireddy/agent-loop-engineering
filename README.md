# Loop Engineering — End-to-End Demo

> **Concept** — [The Batch, Issue 359](https://www.deeplearning.ai/the-batch/issue-359)
> "Loop engineering" is the practice of designing nested feedback loops that let
> AI agents iterate productively on software, with human oversight at the right
> granularity.

This project demonstrates the **3 Key Product Development Loops** as a
runnable Python CLI that builds a Flask To-Do app from scratch using a local LLM
(Ollama).

```
┌──────────────────────────────────────────────────────────────────┐
│  Loop 3 — External Feedback Loop (days)        🟠               │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │  Loop 2 — Developer Feedback Loop (hours)   🟢            │  │
│  │  ┌──────────────────────────────────────────────────────┐  │  │
│  │  │  Loop 1 — Agentic Coding Loop (minutes)  🔵         │  │  │
│  │  │                                                      │  │  │
│  │  │  Spec → LLM generates code → pytest → fix → repeat  │  │  │
│  │  └──────────────────────────────────────────────────────┘  │  │
│  │  Developer reviews → gives feedback → spec updated        │  │
│  └────────────────────────────────────────────────────────────┘  │
│  Users test → feedback collected → vision updated                │
└──────────────────────────────────────────────────────────────────┘
```

## Quick Start

### 1. Prerequisites

- **Python 3.10+**
- **Ollama** running locally ([install guide](https://ollama.com/download))

```bash
# Pull a coding model (any will work — adjust LOOP_ENG_MODEL if needed)
ollama pull qwen2.5-coder:7b
```

### 2. Install

```bash
cd loop-engineering
pip install -e ".[dev]"
```

### 3. Run

```bash
# Initialize the project (creates spec.md + workspace/)
loop-eng init

# Run Loop 1 only — agentic coding
loop-eng code

# Run Loop 1 + 2 — coding + developer feedback
loop-eng dev

# Run all 3 loops end-to-end
loop-eng full

# Check current state
loop-eng status
```

## The Three Loops Explained

| Loop | Command | Timescale | What happens |
|---|---|---|---|
| 🔵 **Agentic Coding** | `loop-eng code` | Minutes | LLM writes code → pytest runs → errors fed back → repeat until tests pass |
| 🟢 **Developer Feedback** | `loop-eng dev` | Hours | You review the app → give natural-language feedback → spec updated → Loop 1 re-runs |
| 🟠 **External Feedback** | `loop-eng full` | Days | Simulated user feedback → LLM synthesizes into spec changes → Loop 2 re-runs |

## Configuration

All settings are driven by environment variables (or `.env` file):

| Variable | Default | Description |
|---|---|---|
| `LOOP_ENG_OLLAMA_URL` | `http://localhost:11434` | Ollama server URL |
| `LOOP_ENG_MODEL` | `qwen2.5-coder:7b` | Model to use |
| `LOOP_ENG_WORKSPACE` | `./workspace` | Where generated code goes |
| `LOOP_ENG_SPEC` | `./spec.md` | Path to the product spec |
| `LOOP_ENG_MAX_AGENTIC_ITER` | `5` | Max code-test-fix cycles |
| `LOOP_ENG_MAX_DEV_ROUNDS` | `10` | Max developer feedback rounds |
| `LOOP_ENG_MAX_EXT_ROUNDS` | `5` | Max external feedback rounds |

## Project Structure

```
loop-engineering/
├── pyproject.toml                   # Project config + deps
├── README.md                        # This file
├── spec.md                          # Product spec (generated by init)
├── feedback.json                    # Optional: pre-written user feedback
├── loop_history.json                # Auto-generated loop history
├── workspace/                       # Agent writes code here
│   ├── app.py                       # (generated) Flask app
│   ├── templates/                   # (generated) HTML templates
│   └── tests/                       # (generated) pytest test suite
└── src/loop_engineering/
    ├── __init__.py
    ├── cli.py                       # Click CLI (init/code/dev/full/status)
    ├── config.py                    # Environment-driven settings
    ├── ollama_client.py             # Ollama REST API wrapper
    ├── prompts.py                   # System prompts for each loop
    ├── display.py                   # Rich console output
    ├── tracker.py                   # Loop history (JSON)
    └── loops/
        ├── __init__.py
        ├── agentic.py               # Loop 1: code → test → fix
        ├── developer.py             # Loop 2: review → feedback
        └── external.py              # Loop 3: user feedback → vision
```
